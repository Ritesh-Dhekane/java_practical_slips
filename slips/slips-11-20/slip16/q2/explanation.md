# Slip 16 – Q2: Method Overloading – Calculator

## What the program does
Defines a `Calculator` class with **5 overloaded `add()` methods** — differing
in number and/or type of parameters. Java picks the correct one at compile time
based on the argument signature (compile-time / static polymorphism).

## Java Concepts Used
- **Method Overloading** – same name, different parameter signatures
- **Compile-time Polymorphism** – resolved at compile time, not runtime
- Handles: `int+int`, `int+int+int`, `double+double`, `int+double`, `String+String`
