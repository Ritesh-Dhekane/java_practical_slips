# Slip 6 – Q1: Exception Handling – NumberFormatException

## What the program does
Accepts a number as a `String` from the user. Tries to parse it with
`Integer.parseInt()`. Catches `NumberFormatException` if the input is
not a valid integer, and always runs the `finally` block.

## Java Concepts Used
- **Exception Handling** – `try`, `catch`, `finally`
- **NumberFormatException** – thrown when string cannot be parsed as integer
- **Integer.parseInt()** – String to int conversion
- **Scanner** – user input
